# 11-源码阅读路线

## 阅读原则

读大型项目不要从页面细节开始。正确顺序是：

```text
先看启动方式
-> 再看路由如何组织页面
-> 再看登录和权限如何拦截
-> 再看请求如何出去
-> 再看状态如何保存
-> 最后进入具体业务页面和复杂组件
```

## 第 1 步：先看项目怎么启动

文件：

- `package.json`
- `vite.config.ts`
- `admin.html`
- `chat.html`
- `src/main.ts`
- `src/chat.ts`

你要学什么：

- 为什么有 `dev` 和 `chat` 两个开发命令。
- Vite 如何通过 `VITE_ENTRY` 决定入口。
- 管理端和聊天端初始化有哪些相同和不同。
- 插件注册顺序：Pinia、Element Plus、指令、router、i18n、全局组件。

这是核心代码，不是业务代码。它决定整个应用如何运行。

## 第 2 步：看路由骨架

文件：

- `src/router/routes.ts`
- `src/router/index.ts`
- `src/router/common.ts`
- `src/router/modules/application.ts`
- `src/router/modules/system.ts`
- `src/router/chat/index.ts`

你要学什么：

- 页面如何被路由组织。
- 路由 meta 如何服务菜单和权限。
- 登录态如何被路由守卫拦截。
- 无权限时如何找替代路由。
- 聊天端为什么有独立守卫。

`system.ts` 是高级前端思维重点，因为它体现复杂权限和企业版能力控制。

## 第 3 步：看用户、登录、权限状态

文件：

- `src/stores/index.ts`
- `src/stores/modules/login.ts`
- `src/stores/modules/user.ts`
- `src/stores/modules/chat-user.ts`
- `src/utils/permission/type.ts`
- `src/utils/permission/index.ts`
- `src/utils/permission/data.ts`
- `src/directives/hasPermission.ts`

你要学什么：

- token 如何保存。
- 登录后为什么还要拉 profile。
- 权限、角色、版本如何组合。
- 路由权限和按钮权限如何复用同一套判断。

这是核心代码。读懂这里，你才能真正理解大型后台的“用户上下文”。

## 第 4 步：看请求封装

文件：

- `src/request/index.ts`
- `src/request/chat/index.ts`
- `src/request/Result.ts`
- `src/api/user/login.ts`
- `src/api/application/application.ts`

你要学什么：

- axios 拦截器怎么注入 token 和语言。
- 为什么管理端和聊天端 request 分开。
- loading、错误、下载、流式请求怎么统一。
- API 层如何隐藏后端 URL 细节。

这是核心代码。业务页面能保持简洁，靠的就是请求层。

## 第 5 步：看布局和菜单

文件：

- `src/layout/layout-template/AppSidelay.vue`
- `src/layout/layout-template/MainLayout.vue`
- `src/layout/layout-template/SystemMainLayout.vue`
- `src/layout/components/sidebar`
- `src/layout/layout-header`

你要学什么：

- 路由 meta 如何变成菜单。
- 当前路由如何决定激活菜单。
- 用户信息、工作空间、语言、主题如何进入布局。

这部分介于核心代码和业务代码之间，是中后台项目很重要的壳层。

## 第 6 步：看组件资产

文件：

- `src/components/index.ts`
- `src/components/dynamics-form/index.ts`
- `src/components/folder-virtualized-tree/VirtualizedTree.vue`
- `src/components/markdown/*`
- `src/components/ai-chat/index.vue`

你要学什么：

- 哪些组件被全局注册。
- 动态表单如何通过组件协议扩展。
- 虚拟树如何解决大数据渲染。
- Markdown 渲染为什么要处理安全。
- AI Chat 如何组合消息、输入、操作按钮。

组件资产体现的是“复用能力”，不是单个页面的业务逻辑。

## 第 7 步：看工作流

文件：

- `src/workflow/common/data.ts`
- `src/workflow/common/validate.ts`
- `src/workflow/common/template.ts`
- `src/workflow/common/edge.ts`
- `src/workflow/plugins/dagre.ts`
- `src/workflow/nodes/*`
- `src/views/application-workflow/index.vue`

你要学什么：

- 节点 type 和节点组件如何对应。
- 节点 properties 里保存哪些配置。
- 工作流边、布局、校验如何拆分。
- 为什么图形化编排要有统一节点协议。

这是项目复杂度最高的模块，建议最后读。

## 哪些是核心代码

- 入口：`src/main.ts`、`src/chat.ts`
- 路由：`src/router/*`
- 请求：`src/request/*`
- 状态：`src/stores/modules/user.ts`、`login.ts`、`chat-user.ts`
- 权限：`src/utils/permission/*`
- 工程：`vite.config.ts`
- 复杂组件：`dynamics-form`、`ai-chat`、`workflow`

## 哪些只是业务代码

- `src/views/application`
- `src/views/knowledge`
- `src/views/document`
- `src/views/model`
- `src/views/tool`
- `src/views/system`

注意：“只是业务代码”不是不重要，而是它们更适合在你掌握架构骨架后再读。否则你会陷在表单、弹窗、字段里，看不到工程思想。

## 哪些体现高级前端思维

- `import.meta.glob` 自动收集模块。
- 双入口隔离管理端和聊天端。
- 多维权限模型。
- request 层统一基础设施。
- workspace 动态 API 前缀。
- 虚拟树处理长列表。
- Markdown XSS 白名单。
- 工作流节点协议。

## 推荐阅读节奏

第一周：入口、路由、请求、状态、权限。

第二周：布局、菜单、全局组件、动态表单。

第三周：应用、知识库、文档三个业务域，各挑一个页面从 UI 追到 API。

第四周：AI Chat 和 Workflow，重点理解复杂组件如何拆协议。
