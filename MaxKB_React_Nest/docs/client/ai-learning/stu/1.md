# package.json

```
 "build": "run-p type-check \"build-only {@}\" --",
    "build-chat": "run-p type-check \"build-only-chat {@}\" --",
```

这两个脚本里的“插件”其实是 npm-run-all2 提供的命令，不是 Vite 插件。

意思大致是：

```
1. pnpm type-check
2. pnpm build-only
```

{@} 是 npm-run-all2 的参数占位符，用来把你传给 build 的额外参数继续传给 build-only 或 build-only-chat。比如你执行：

```
pnpm build -- --mode production
```

# vite.config.ts

```
1. 读取环境变量
2. 组装代理规则
3. 注册 Vite 插件
4. 配置开发服务器
5. 配置构建和路径别名
```

构建后把 admin.html 或 chat.html 重命名成 index.html。

```ts
renameHtmlPlugin(...)
```

用于把 /admin/api、/chat/api、/doc、/schema、/static 等请求代理到后端。

```ts
server.proxy
```

指定环境变量目录，不用默认根目录 .env。

```ts
envDir: './env'
```

根据当前模式读取环境变量，同时额外读取聊天端端口。

```ts

- mode：环境模式（如 development/production/chat 自定义名）
- envDir：环境文件所在的目录

// ENV：加载 .env、.env.development、.env.production 等默认环境文件
const ENV = loadEnv(mode, envDir)
// CHAT_ENV：专门加载 .env.chat 这个自定义环境文件
const CHAT_ENV = loadEnv('chat', envDir)
```

根据环境变量决定使用哪个 HTML 入口，比如 admin.html 或 chat.html。

```ts
createHtmlPlugin({ template: ENV.VITE_ENTRY })
```

告诉 Vite/Rollup 构建哪个入口文件。

```ts
rollupOptions: {
  input: ENV.VITE_ENTRY,
}
```

根据部署路径决定构建产物输出目录。

```ts
outDir: `dist${ENV.VITE_BASE_PATH}`
```

让静态资源使用相对路径，适配非根路径部署。

```ts
base: './'
```

把 chat 模式下的端口注入到运行时代码里。admin跳转chat用到

```ts
define: {
  'import.meta.env.VITE_CHAT_APP_PORT': JSON.stringify(CHAT_ENV.VITE_APP_PORT),
}
```
